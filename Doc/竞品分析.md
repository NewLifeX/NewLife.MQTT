# NewLife.MQTT 竞品分析

> **分析日期**：2026 年 3 月
> **对标版本**：NewLife.MQTT 2.3.x / EMQX 5.x / Mosquitto 2.x / HiveMQ 4.x / VerneMQ 2.x / NanoMQ 0.x / MQTTnet 4.x / Eclipse Paho C# 1.x

---

## 1. 竞品全景

### 1.1 竞品分类

MQTT 生态产品主要分为三大类：

| 分类 | 产品 | 定位 |
|------|------|------|
| **全功能 Broker** | EMQX、HiveMQ、VerneMQ | 企业级 MQTT 服务端，多语言客户端 SDK 另行提供 |
| **轻量级 Broker** | Mosquitto、NanoMQ | 嵌入式/边缘场景，功能精简 |
| **.NET 生态库** | MQTTnet、Eclipse Paho C# | .NET 专属客户端库 |
| **一体化协议库** | **NewLife.MQTT** | 客户端 + 服务端 + 企业功能一体，.NET 全版本 |
| **云托管服务** | AWS IoT Core、Azure IoT Hub、阿里云微消息队列 MQTT 版 | 全托管 MQTT 服务，按量计费 |

## 2. 服务端功能对比

### 2.1 协议支持

| MQTT 5.0 | ✅ | ✅（v2.0+） | ✅ | ✅ | ✅ | ✅（30/30 属性） |

**分析**：

- NewLife.MQTT 在传输协议覆盖面上**业界最广**（6 种），是唯一同时支持 QUIC、可靠 UDP 和 ProxyProtocol 的实现

- **MQTT over 可靠 UDP 为全球独创**，可覆盖 ARM Linux 4.x 等不支持 msquic 的嵌入式平台
- 不支持 MQTT-SN 是有意的边界限定（见需求文档 §5.1）

### 3.2 API 易用性对比

**MQTTnet — 连接与订阅**

```csharp
// MQTTnet：需要 Builder 模式，配置项分散
var factory = new MqttFactory();
var client = factory.CreateMqttClient();
var options = new MqttClientOptionsBuilder()
    .WithTcpServer("broker.example.com", 1883)
    .WithCredentials("admin", "password")
    .WithClientId("myDevice")
    .Build();

await client.ConnectAsync(options);

// 订阅写法繁琐，且断线重连后需手动重新订阅
await client.SubscribeAsync(new MqttTopicFilterBuilder()
    .WithTopic("/test/#")
    .WithQualityOfServiceLevel(MqttQualityOfServiceLevel.AtLeastOnce)
    .Build());

// 消息回调需要另外注册
client.ApplicationMessageReceivedAsync += e =>
{
    Console.WriteLine(Encoding.UTF8.GetString(e.ApplicationMessage.PayloadSegment));
    return Task.CompletedTask;
};
```

**NewLife.MQTT — 同等功能**

```csharp
// NewLife.MQTT：极简 API，一行订阅含回调
var client = new MqttClient
{
    Server = "tcp://broker.example.com:1883",
    UserName = "admin",
    Password = "password",
    ClientId = "myDevice",
};
await client.ConnectAsync();

// 订阅即绑定回调，重连自动重订阅
await client.SubscribeAsync("/test/#", e =>
    XTrace.WriteLine($"{e.Topic}: {e.Payload.ToStr()}"));
```

**或更简洁的连接字符串方式**

```csharp
var client = new MqttClient();
client.Init("Server=tcp://broker.example.com:1883;UserName=admin;Password=password;ClientId=myDevice");
await client.ConnectAsync();
```

**对比总结**：NewLife.MQTT 的 API 更加简洁直观。订阅时直接绑定回调（一行完成），无需 Builder 模式和分离的事件注册。连接字符串初始化支持配置中心场景。

## 4. 性能对比

### 4.1 已知性能数据

| 产品 | 单机连接数 | 消息吞吐 | 数据来源 |
|------|-----------|---------|---------|
| **EMQX** | 500 万（官方宣称） | 100 万+ msg/s（集群） | 官方文档、第三方评测 |
| **Mosquitto** | 10 万级 | 20 万 msg/s（单核） | 社区评测 |
| **HiveMQ** | 千万级（企业版） | 数百万 msg/s（集群） | 官方白皮书 |
| **VerneMQ** | 百万级 | 100 万+ msg/s | 官方 benchmark |
| **NanoMQ** | 10 万级 | 100 万+ msg/s | 官方文档（边缘场景） |
| **MQTTnet** | 万级（嵌入式服务端） | 10 万 msg/s | 社区反馈 |

> **注**：NewLife.MQTT 的高性能得益于底层 NewLife.Net 网络库的极致优化（IO 多路复用、零拷贝、对象池、缓冲区复用），2266 万 tps 为网络层基准数据。

### 4.2 性能架构差异

| 维度 | EMQX | Mosquitto | MQTTnet | **NewLife.MQTT** |
|------|------|-----------|---------|-----------------|
| 并发模型 | Erlang Actor | 单线程轮询 | async/await（ThreadPool） | **NewLife.Net 事件驱动 + IO 多路复用** |
| 内存管理 | Erlang GC | 手动管理 | .NET GC | **对象池 + ArrayPool + Span** |
| 网络层 | Erlang Port | epoll/kqueue | System.Net.Sockets | **NewLife.Net（自研高性能网络层）** |
| 序列化 | Erlang Binary | 手写二进制 | 手写二进制 | **手写二进制 + Pool.StringBuilder** |
| 定时器 | Erlang Timer | 系统定时器 | System.Threading.Timer | **TimerX 高精度定时器** |

## 8. SWOT 分析

### 8.1 Strengths（优势）

| 优势 | 详细说明 |
|------|---------|
| **零依赖、无供应链风险** | 仅依赖 NewLife.Core，NuGet 依赖链中无第三方包，无版本冲突和安全漏洞传播风险 |
| **客户端+服务端一体交付** | 全球唯一单一 NuGet 包同时提供全功能客户端和服务端，极简集成 |
| **六种传输协议** | TCP/TLS/WS/WSS/QUIC/可靠UDP，覆盖面业界最广，其中 QUIC 业界罕见，可靠 UDP 全球独创 |
| **企业功能免费内置** | 桥接/规则引擎/WebHook/统计，竞品需企业版付费或插件扩展 |
| **全版本 .NET 覆盖** | net45 ~ net10.0，跨越 15 年 .NET 运行时，兼容性最广 |
| **API 极简** | 订阅即回调、连接字符串初始化、自动重订阅，开发体验优于竞品 |
| **高性能** | 基于 NewLife.Net，已验证 400 万连接 + 2266 万 tps |
| **信创合规** | 纯国产自主研发 + MIT 开源 + 零外部依赖 = 无法律和供应链风险 |
| **断线自动重订阅** | 竞品（MQTTnet/Paho）需手动重订阅，NewLife.MQTT 自动完成 |

### 8.2 Weaknesses（劣势）

| 劣势 | 影响 | 改进方向 |
|------|------|---------|
| 持久化仅内存级 | 服务重启后丢失会话和离线消息 | 实现 Redis/SQLite 持久化的 `IMqttExchange` |
| 缺少 HTTP 管理 API | 无法通过 API 管理 Broker 状态 | 基于 WebApi 添加 RESTful 接口 |
| 缺少 Dashboard | 无可视化管理界面 | 对接 Grafana 或自建 Web UI |
| 集群自动发现缺失 | 扩缩容需手动配 | 集成 Stardust / mDNS / Consul |
| 社区生态较小 | 文档和社区帮助有限 | 持续完善文档，扩大社区影响力 |
| MQTT 5.0 少量功能待完善 | 遗嘱延迟、消息过期、SASL | 按需求文档迭代实现 |

### 9.3 结论

NewLife.MQTT 在 MQTT 产品市场中占据一个**独特的细分位置**：

- **vs MQTTnet**：不仅是客户端库，同时提供全功能服务端 + 企业功能，覆盖的传输协议更多
- **vs Mosquitto**：更适合 .NET 技术栈，企业功能免费内置，不需要额外部署 C 语言服务端

**核心价值主张**：_对于使用 .NET 技术栈的 IoT 项目，特别是有信创合规要求的中大型企业，NewLife.MQTT 是唯一能在单一零依赖 NuGet 包中同时获得完整客户端、高性能服务端和企业级功能的选择。_
